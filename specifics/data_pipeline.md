# Post-run data pipelines

The canonical event log stores semantic simulation events for post-run
analysis. It is not a graph format. G0/G1/G2/G3 are projections derived from
this log plus state and spatial snapshots.

## Files

- `events_rank<N>.jsonl`: rank-local semantic events.
- `events.jsonl`: merged semantic events - merged by rank0.
- `event_log_metadata.json`: schema version and event type notes.
- `spatial_snapshots*.jsonl`: spatial state snapshots for post-run analysis.
- `tick_metrics.csv` / exported metric tables: compact scalar and count series.

`python main.py export-data` can normalize the run logs into table-oriented
CSV/JSONL files under `output/tables` by default:

- `events`
- `event_effects`
- `spatial_snapshots`
- `tick_metrics`
- `agent_counts_by_tick`

The graph builders consume the semantic logs and write only:

- `g0.gexf`
- `g0.lite.gext`
- `g1.gexf`
- `g2.gexf`
- `g3.gexf`

No graph JSON, manifest or report files are generated by the graph builders.

## Event Types

- `state_transition`: one agent changed biological/model state.
- `threshold_trigger`: one observed field/condition triggered a state update.
- `action_selected`: one agent selected an action.
- `field_change`: one agent changed one scalar field by a signed delta.
- `aggregation`: one alpha-synuclein aggregation operation resolved.
- `degradation`: one lysosome degradation operation resolved.

The schema intentionally excludes:

- `buffer_committed`: buffers are runtime implementation details.
- `target_assignment`: target queues are runtime implementation details; the
  canonical log records degradation when it resolves.
- graph-only identifiers such as G0 node IDs, G1 keys, G2 keys or GEXF labels.

## Common Shape

```json
{
  "schema_version": "1.0-canonical-event",
  "run_id": "default_run",
  "event_id": "event_0_12_00000034",
  "tick": 12,
  "phase": "4_effect_application",
  "rank": 0,
  "event_type": "field_change",
  "mechanism": "microglia_debris_clearance",
  "rule_id": "MICROGLIA_CLEARING_DEBRIS_PRESSURE",
  "actor": {
    "uid": "3:1:0",
    "type": "Microglia",
    "state": "Clearing",
    "compartment": "Extracellular"
  },
  "effects": [
    {
      "kind": "field_delta",
      "scope": "environment",
      "field": "extracellular_debris",
      "delta": -0.05,
      "sign": "negative",
      "unit": "normalized"
    }
  ],
  "outcome": "applied"
}
```

## Design Rule

Each event should answer:

- who acted;
- what changed;
- where and when it happened;
- which mechanism/rule caused it;
- what the outcome was.
